# PRD: 공공조달관리사 필승 100% 합격 학습 시스템
## Product Requirements Document v1.0

---

## 1. 제품 개요

### 1.1 제품명
**공공조달관리사 필승 합격 시스템** (PPMS: Public Procurement Manager Study System)

### 1.2 제품 비전
2026년 최초 시행되는 공공조달관리사 국가기술자격 시험의 필기(객관식 80문항) 및 실기(필답형)를 **단시간에 핵심 집중 학습**하여 **100% 합격**을 달성하는 AI 기반 적응형 웹 학습 시스템

### 1.3 핵심 가치 제안
- 기출문제가 **전무한 최초 시행 시험**에 대해 출제기준 기반 예측 학습
- 표준교재 4권(약 1,200페이지) + 출제기준 180개 세세항목을 **체계적으로 압축**
- AI 기반 **적응형 학습 경로** 제공으로 취약 영역 자동 보강
- 에빙하우스 망각곡선 기반 **과학적 반복 학습**

### 1.4 대상 사용자
| 사용자 유형 | 설명 | 핵심 니즈 |
|---|---|---|
| **1차 타겟** | 공공조달 실무자 (조달청, 수요기관, 공공기관) | 업무 경험 기반 체계적 정리, 실기 서술형 훈련 |
| **2차 타겟** | 공공조달 업체 종사자 (입찰·계약 담당) | 법령·제도 체계적 학습, 실무 적용 역량 확보 |
| **3차 타겟** | 자격 취득 희망자 (취업준비생, 전직 희망자) | 기초부터 체계적 학습, 단기 합격 전략 |

## 6. 데이터 모델 (핵심 엔티티)

```
User (사용자)
├── id, email, nickname, created_at
├── study_plan_start_date
├── target_exam_date (2026-10-03)
└── current_phase (1~4)

Subject (과목)
├── id, name, question_count, weight_percent
└── textbook_volume (1~4)

MainTopic (주요항목)
├── id, subject_id, name, order
└── estimated_questions, estimated_weight

SubTopic (세부항목)
├── id, main_topic_id, name, order
└── detail_items[] (세세항목 배열)

DetailItem (세세항목)
├── id, sub_topic_id, name
├── prediction_score (출제확률 1~5)
├── cross_subject_links[] (과목 간 연계)
└── practical_exam_links[] (실기 연계)

ConceptCard (개념카드)
├── id, detail_item_id, front_text, back_text
├── category (개념/비교/숫자/법조문/절차)
├── difficulty (기초/응용/실전)
└── review_interval (에빙하우스 간격)

Question (문제)
├── id, detail_item_id, subject_id
├── question_text, options[4], correct_answer
├── difficulty (1~3), explanation
├── wrong_explanations[4] (오답해설)
└── tags[]

MockExam (모의고사)
├── id, user_id, created_at
├── questions[] (80문항 배열)
├── answers[], score_by_subject{}
└── total_score, time_spent

UserProgress (학습 진도)
├── user_id, detail_item_id
├── concept_card_mastered (boolean)
├── question_correct_rate (0~100)
├── last_reviewed_at, next_review_at
└── review_count

PracticalAnswer (실기 답안)
├── id, user_id, scenario_id
├── answer_text, ai_score
├── feedback_text, model_answer_diff
└── star_l_structure_score
```

## 7. 개발 로드맵

### Phase 1: MVP (Week 1~4) — 핵심 학습 기능
- [ ] 프로젝트 셋업 (Next.js + Supabase + Prisma)
- [ ] 인증 시스템 (이메일/소셜 로그인)
- [ ] 출제기준 DB 구축 (170개 세세항목 입력)
- [ ] 개념카드 학습 모듈 (에빙하우스 반복)
- [ ] 기본 문제풀이 (과목별 기초 문항 100개)
- [ ] 대시보드 (학습 진도, D-Day)

### Phase 2: 적응형 학습 (Week 5~8)
- [ ] 적응형 문제은행 확장 (과목별 300문항+)
- [ ] 난이도 자동 조절 알고리즘
- [ ] 취약 영역 자동 진단 및 보강 추천
- [ ] 비교표/숫자 암기 퀴즈 모드
- [ ] 오답노트 및 북마크 기능

### Phase 3: 실전 모의고사 (Week 9~12)
- [ ] 실전 모의고사 모드 (80문항/2시간)
- [ ] 자동 채점 + 영역별 분석 리포트
- [ ] 성적 추이 그래프
- [ ] 합격 예측 기능
- [ ] 출제확률 히트맵

### Phase 4: 실기 코칭 + 최적화 (Week 13~16)
- [ ] 실기 서술형 문제 시나리오
- [ ] AI 채점 모듈 (OpenAI 연동)
- [ ] STAR-L 답안 구조화 가이드
- [ ] PWA 오프라인 지원
- [ ] 성능 최적화 및 최종 테스트
