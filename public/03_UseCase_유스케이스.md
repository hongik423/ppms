## 1. 액터(Actor) 정의

| 액터 | 역할 | 설명 |
|---|---|---|
| **학습자(User)** | 주 액터 | 시스템에 가입하여 학습·문제풀이·모의고사를 수행하는 수험생 |

| **관리자(Admin)** | 부 액터 | 문제 등록·수정, 콘텐츠 관리, 사용자 통계 조회 (Phase 2+) |

### UC-01: 회원가입

| 항목 | 내용 |
|---|---|
| **액터** | 학습자 |
| **선행조건** | 없음 |
| **트리거** | 학습자가 /auth/register 접근 |

**기본 흐름:**
1. 학습자가 이메일/비밀번호 입력 (또는 소셜 로그인 선택)
2. 시스템이 이메일 중복 검사 수행
3. Supabase Auth로 계정 생성
4. 닉네임, 시험일(기본값 2026-10-03) 입력 화면 표시
5. User 레코드 생성 (current_phase=1, study_plan_start_date=오늘)
6. 진단평가 안내 화면으로 이동

**대안 흐름:**
- 3a. 이메일 중복 → 오류 메시지 "이미 가입된 이메일입니다"
- 1a. 소셜 로그인(Google/Kakao) → OAuth 흐름 후 5번으로

**후행조건:** User 테이블에 신규 레코드 생성, 대시보드 초기 상태 설정

- 로그아웃: 세션 파기 → /auth/login 이동

### UC-03: 초기 진단평가

| 항목 | 내용 |
|---|---|
| **액터** | 학습자, 시스템 |
| **선행조건** | 최초 가입 후 진단평가 미완료 |
| **트리거** | 가입 직후 또는 대시보드에서 "진단평가 시작" 클릭 |

**기본 흐름:**
1. 시스템이 과목별 20문항 (총 60문항) 진단 문제 출제
2. 문항별 세세항목 태그가 전체 출제기준을 균등 커버
3. 학습자가 과목 선택 → 해당 과목 20문항 풀이 (시간제한 없음)
4. 풀이 완료 시 자동 채점
5. 세세항목별 정답률 산출 → UserProgress 레코드 일괄 생성
6. 취약 영역 분석: 정답률 50% 미만 항목 식별
7. 결과 요약 화면 표시 (과목별 정답률, 취약항목 TOP 5)
8. 시스템이 D-Day 기반 맞춤 학습 계획 자동 생성

**대안 흐름:**
- 3a. 과목 일부만 진단 → 완료 과목만 분석, 미완료 과목은 "진단 필요" 표시
- 8a. D-Day 200일 이상 → Phase 1부터 순차 배분
- 8b. D-Day 60일 이하 → Phase 3(실전훈련)에 학습 역량 집중 배분

**후행조건:** UserProgress 테이블에 ~170개 세세항목 초기 데이터 생성, 학습계획 생성

### UC-04: 개념카드 학습 (에빙하우스 반복)

| 항목 | 내용 |
|---|---|
| **액터** | 학습자, 시스템 |
| **선행조건** | 로그인 완료 |
| **트리거** | /learn/cards/today 접근 또는 대시보드 "오늘의 복습" 클릭 |

**기본 흐름:**
1. 시스템이 에빙하우스 주기에 따른 오늘의 복습 카드 목록 생성
   - 신규 카드: 출제 확률 높은 세세항목 우선 배정
   - 복습 카드: next_review_at ≤ 오늘인 카드 자동 포함
2. 카드 앞면(질문/키워드) 표시
3. 학습자가 탭하여 뒷면(정답/해설/법조문) 확인
4. 학습자가 자기평가 수행:
   - 😰 어려움 → 다음 복습 1일 후
   - 🤔 보통 → 다음 복습 3일 후 (또는 현재 간격 유지)
   - 😊 쉬움 → 다음 복습 간격 2배 (3→7→14→30일)
5. UserProgress.next_review_at 업데이트
6. 모든 카드 완료 시 결과 요약 (총 N장, 어려움/보통/쉬움 비율)

**대안 흐름:**
- 1a. 복습할 카드가 0장 → "오늘은 모든 복습을 완료했습니다. 새 카드를 학습하시겠습니까?"
- 4a. 3회 연속 "쉬움" 평가 → concept_card_mastered=true로 변경

**후행조건:** UserProgress.review_count 증가, next_review_at 갱신

**비즈니스 룰:**
- 에빙하우스 간격: 1일 → 3일 → 7일 → 14일 → 30일
- 일일 신규 카드 상한: 20장 (설정에서 조정 가능)
- 마스터 기준: 3회 연속 "쉬움" 평가

### UC-10: 빠른 문제풀이

| 항목 | 내용 |
|---|---|
| **액터** | 학습자, 시스템 |
| **선행조건** | 로그인 완료 |
| **트리거** | /practice/quick 접근 |

**기본 흐름:**
1. 학습자가 설정 선택:
   - 과목: 1과목 / 2과목 / 3과목 / 전체
   - 주요항목: 전체 / 특정 항목 선택
   - 난이도: 기초 / 응용 / 실전 / 혼합
   - 문항 수: 10 / 20 / 30
2. 시스템이 조건에 맞는 문제 랜덤 추출 (이전 오답 문제 우선 포함)
3. 문제 화면 표시: 문항번호, 과목/항목 라벨, 4지선다
4. 학습자가 답 선택
5. 즉시 채점 (정답/오답 표시)
6. 해설 패널 표시:
   - 정답 해설 (왜 맞는지)
   - 오답 해설 (각 선지가 왜 틀린지)
   - 관련 법조문/교재 페이지
7. 다음 문항으로 이동
8. 전체 완료 시 결과 요약: 정답률, 소요시간, 취약항목

**대안 흐름:**
- 4a. 북마크 버튼 클릭 → 해당 문제 북마크 테이블에 저장
- 8a. 정답률 50% 미만 → "개념 복습을 추천합니다" + 해당 개념카드 링크

**후행조건:** UserProgress.question_correct_rate 갱신, 풀이 이력 저장

### UC-11: AI 맞춤 문제풀이

| 항목 | 내용 |
|---|---|
| **액터** | 학습자, 시스템 |
| **선행조건** | 진단평가 완료 또는 문제풀이 이력 10문항 이상 |
| **트리거** | /practice/adaptive 접근 |

**기본 흐름:**
1. 시스템이 UserProgress 분석:
   - 출제확률 높은데 정답률 낮은 세세항목 우선 추출
   - 가중치 공식: 추천점수 = 출제확률(1~5) × (100 - 정답률) / 100
2. 상위 추천점수 항목에서 20문항 자동 출제
3. 난이도 적응형 조절:
   - 최근 5문항 정답률 80%↑ → 다음 문항 난이도 상향
   - 최근 5문항 정답률 50%↓ → 다음 문항 난이도 하향
4. 풀이 흐름은 UC-10의 3~8과 동일

**비즈니스 룰:**
- 추천 문제는 최근 7일 내 풀지 않은 문제 우선
- 동일 세세항목에서 연속 3문항 이상 출제하지 않음

## 5. 비기능 유스케이스

### NF-01: 오프라인 학습 지원 (PWA)
- 개념카드, 문제풀이는 오프라인 캐싱 지원
- 오프라인 학습 데이터 → 온라인 복귀 시 동기화
- Service Worker로 정적 에셋 + 최근 학습 데이터 캐싱

### NF-02: 반응형 UI 적응
- 데스크톱 (1200px+): 사이드바 + 메인 콘텐츠 레이아웃
- 태블릿 (768~1199px): 접이식 사이드바
- 모바일 (767px 이하): 하단 탭 바, 전체 화면 카드 UI

### NF-03: 데이터 자동 저장
- 모의고사 진행 중 답안: 문항 선택 시 즉시 LocalStorage 저장
- 실기 답안 작성 중: 30초 간격 자동 임시 저장
- 학습 진도: API 호출 성공 시 서버 동기화, 실패 시 큐잉 후 재시도

### NF-04: 접근성
- 키보드 네비게이션 (Tab/Enter/방향키로 모든 기능 접근)
- 스크린 리더 호환 (aria-label, role 설정)
- 색맹 대응 (색상 외 아이콘/텍스트 병기)
